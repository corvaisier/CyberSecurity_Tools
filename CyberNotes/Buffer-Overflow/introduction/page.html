Dans la Stack différents registres :
ESP (Extended Stack Pointer)
Buffer space
EBP (Extended Base Pointer)
EIP (Extended Instruction Pointer) / Return address

Si buffer overflow attack : être caapble de déborder sur buffer space, de l'EBP pour écrire sur le EIP qui retourne des adresses donc lui demander de retourner sur un reverse shell.

Plusieurs étapes :
1 - Spiking : Trouver si le programme est vulnérable
2 - Fuzzing, autre méthode similaire
3 - Trouver l'offset
4 - Réécrire le EIP
5 - Trouver les caractères interdits
6 - Trouver le bon module
7 - Générer un shellcode
8 - Root !
