Active Directory :

Annuaire de tous les machines : Imprimantes....
Tout est objet : tel qu'un ordinateur, personne..
Tout ce qui est authentification passe par Kerberos

Les composants physiques :
Domain Controller => beaucoup de rôles. Il héberge l'annuaire, il gère l'authentification et l'autorisation.
  • Il réplique d'autres controlleurs de domaine, et il a un accès pour l'administration.
AD DS data store => il contient toute la base de donnée du controlleur de domaine, avec toutes les informations des utilisateurs, services...

Les composants logiques :
AD DS schema => Définit tous les objets possibles qui peuvent être stockés dans l'AD, et définit les règles de configuraton.
Domains => La façon de grouper les objets entre eux. Il peut n'y avoir qu'un domaine pour des petits commerces. Chaque domaine a son Domain Controller.
Trees => Il est composé de plusieurs domaines, il peut ^avoir un Domain parent, et plusieurs domaines enfants. Ils ont un lien de confiance entre eux.
Forests => Une collection de plusieurs Trees.
Organizational Units (Ous) => une unité d'organisation
Trusts => C'est le mecanisme qui permet d'accorder l'acces à des ressources pour un autre domaine. Il peut être directionnel ou multidirectionnel
Objects => A l'interieur d'un Ous, Ce peut être un utilisateur, une imprimante ou un ordinateur.
